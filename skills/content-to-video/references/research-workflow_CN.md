# 研究工作流程

内容转视频深度研究阶段的详细指南。

## 内容源处理

### URL 和文章

1. **获取内容**：
   - 如果用户没有提供具体的 URL 和文章链接，需要深度探索各种平台，使用 WebSearch 收集支撑材料
   - 使用 WebFetch 工具提取主要内容
   - 移除广告、导航、页脚
   - 提取作者、日期、出版物

2. **关键信息**：
   - 主要论点/论题
   - 支撑论点（3-5 个要点）
   - 引用和数据
   - 图片和图表

3. **保存到** `deep-research/content/`：
   - `article.md` - 清理后的 markdown
   - `metadata.json` - 标题、作者、URL、日期
   - `highlights.md` - 关键摘录

### 学术论文

1. **解析 PDF**：
   - 提取文本和结构
   - 识别章节：摘要、引言、方法、结果、结论

2. **关键信息**：
   - 研究问题
   - 方法摘要
   - 主要发现（3-5 点）
   - 图表和插图
   - 引用

3. **可视化**：
   - 将图表重构为 SVG/React 组件
   - 生成解释性图表
   - 截取重要图片

4. **保存到** `deep-research/content/`：
   - `paper.md` - 提取的文本
   - `summary.md` - 浓缩摘要
   - `figures/` - 提取的图片

### GitHub 仓库

1. **分析结构**：
   - README 内容
   - 主要代码文件
   - 包依赖
   - 架构模式

2. **关键信息**：
   - 项目目的
   - 主要功能（3-5 个）
   - 代码示例
   - 安装/使用方法

3. **可视化**：
   - 目录树图
   - 架构图
   - 代码流程图

4. **保存到** `deep-research/content/`：
   - `readme.md` - README 内容
   - `analysis.md` - 代码分析
   - `examples/` - 代码片段

## 资源收集

### 来源

### 额外资源生成和创建

- 使用 AgentBrowser 浏览网页（GitHub、产品主页、论文），从源页面录制视频（滚动到底部，或选择重要文本），详见 agent-browser 技能
- 可以使用脚本从论文或网页截取重要图表并保存，或下载相关视频
- 搜索 x.com 帖子、小红书、Reddit 等平台的评论，生成对应的 HTML 或 SVG 转换为图片
- 可以生成各种 SVG 动画供视频使用，如公式、图表、Lottie 等
- 使用 Mermaid、Excalidraw 进行解释说明

## BGM 研究

### 来源

- BGM 选项（搜索和下载，详见 free-resource 技能）

### 选择标准

1. **情绪**：匹配视频基调（欢快、平静、戏剧性）
2. **节奏**：与配音节奏一致
3. **时长**：或可循环的片段
4. **授权**：如需要，验证商业使用权限

### 保存到 `deep-research/bgm/`：
```
bgm/
├── track-1.mp3
├── track-2.mp3
└── options.json
```

## Token 管理

**问题**：研究可能消耗大量 token。

**解决方案**：
1. **增量保存**：频繁写入文件
2. **摘要**：尽早创建浓缩版本
3. **选择性加载**：只加载当前步骤所需的内容
4. **外部工具**：使用 Task 代理进行并行研究

## 质量检查清单

进入会话规划前：
- [ ] 主要内容已提取并保存
- [ ] 已识别 3-5 个关键要点
- [ ] 已收集 10+ 张潜在图片
- [ ] 有 2-3 个 BGM 选项
- [ ] 研究笔记已完成
- [ ] 目标受众已定义
- [ ] 视频风格已确定
